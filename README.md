# SaaS Web前端新手指引

### 1.日报
SaaS组成员需要每天写日报，[日报规范](./【SaaS组日周月报规范要求-2017.04.21】.pdf)。

日报标题格式为：【会分期】日报_SaaS组_xxx

日报收件人：saas-rd

日报正文分为3部分：

1.工作目标

2.今天工作内容

3.明天工作计划

如果当天工作存在某些问题或者某些原因会造成项目延期需要添加“目前存在问题”一项。

### 2.代码规范、git提交规范
[代码/文件命名规范](https://github.com/hfq-zhangpeng/standard)

关于git提交，在进行commit时需要填写相应提交的备注。
建议少量、多次进行git提交。
